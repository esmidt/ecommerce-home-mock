Assignment: Building Pages with HTML and CSS

In this assignment, you'll build two websites from scratch. The goal is to get you warmed up and comfortable with the basics of HTML and CSS. It takes a lot of experience to get over the "fighting with CSS" stage of things so don't worry if you still end up frequently Googling for selectors.

Remember to use your browser's Developer Tools and move on if you're hung up on a particular issue. You'll get a lot more chance to work with CSS during the larger project which follows this assignment.

You'll also get a chance to jump back into Git, the version control system you learned during the Web Development Basics mini-course. If you need a refresher, check out the Lesson on Git.

Warmup: Google.com Homepage

Your first task is to build a clone of Google.com's homepage, a classic project designed to get you thinking in markup. If you've never built a real webpage before, this should be an exciting introduction!

Before getting started, do some stretches by making sure you know:

  1. Two ways to move a div around on the page
  2. Stick a div onto the bottom or top of the page
  3. Identify the background color of an existing webpage
  4. Grab the URL for an image from an existing webpage
  5. Center an element horizontally
  6. Identify three ways you can include your CSS styles in a page
  7. Understand how to use classes and ids to target CSS at specific elements on the page
  8. Build a very basic form (even if it doesn't "go" anywhere)

All set? Great. Now you need to set up your Git repo for the project. If you recall, the basic process goes something like this (there are a couple of different ways to do it):

  1. Create a new project on Github
  2. git clone that project onto your local machine
  3. cd into the folder with your project and start working, being sure to git add and git commit often.
  4. git push origin master when you're ready to put it up onto Github.

For this project, you'll want to create an index.html file and link to your CSS file from within the HTML file's <head> tag.

Your Task

Clone the Google.com homepage! You don't need to make it pixel perfect and the form doesn't need to submit anywhere... just focus on getting all the elements in approximately the right spots. By the same token, you don't need to worry about the little details like dropdown menus or hover effects (unless you want to of course).

  1. Start with just putting the main elements on the page (the logo image and search form), then get them placed horizontally. You can either download the Google logo or link directly to its URL on the web in your <img> tag.
  2. Next do the navbar across the top, first building the content and then trying to position it. Check out how to build a horizontal CSS navbar if you're lost.
  3. Finally, put in the footer, which should be very similar to the top navbar.
  4. In general, do as much on your own as you can before relying on the developer tools (or viewing the page's source code) to help you along.
  5. If you're frustrated with trying to get buttons or inputs to style the way you want (for instance, they seem to just not respond to any styles), look into the css property -webkit-appearance: none; or -moz-appearance if you're using Firefox.
  6. Push your project to Github when you're ready.

Pro Tip: If you didn't know before, there is actually a way to display an HTML file directly from Github by using a link of the form: 

http://htmlpreview.github.io/?https://github.com/YOUR_USERNAME/PATH/TO/YOUR/index.html

The Viking Store E-Commerce App Homepage

Remember the Viking Store e-commerce app you broke into stories and added a pagination feature to in the mini-course on Engineering? Well, they've decided to rebuild the app now that they know more about what people want.

The Product Manager and Designer have provided you with the mockup and some assets for the site. Your job is to turn the mockup above of the Viking Store home page into a valid HTML page.

The designer also noted that the main background image at the top should be faded and unobtrusive (the asset provided is solid black and white). The assets you need are included below.

Optional Extensions

If you'd like some more practice with Creating Mockups, start by recreating the mockup above or even coming up with your own template for a simple e-commerce website.

If you'd like some more practice with Agile User Stories, first break the mockup apart into stories and put those into Pivotal Tracker.

Your Basic Process

Your basic process will be similar to the Google Homepage above, with a few extra steps:

  1. Break the mockup into definite pieces of work (e.g. the navbar vs main product area). This is where user stories are useful...
  2. Prioritize those pieces of work / stories.
  3. Create a Github repo for the project and pull it down to a local repo.
  4. Start building with HTML/CSS, working through each story/piece of work.
  5. Save and upload to Github.
